# Bash calls
Prefer discrete, single-purpose Bash calls over one `A && B && C` chain. Reserve `&&` for a true dependency (B must not run if A failed) — never to staple unrelated steps. Run independent steps as separate calls (parallel in one message when they don't depend on each other). Never put a known-prompting or sensitive step in a chain or a parallel batch — a prompt cancels its siblings; give it its own call. No confirmation padding: drop trailing `echo`/`pwd`/`ls`/`env | grep` — read the real command's output and trust the env (don't re-verify vars). Genuinely multi-step routines belong in an allowlisted helper script (`bin/*`), not an inline mega-chain. Run `gh`/`git` from the repo or worktree cwd — never `cd ~` (home) first: home is not a git repo, so the call (lacking `--repo`/`-C`) dies `not a git repository`. If you can't stay in-tree, pass `gh --repo <owner/name>` or `git -C <repo>` instead of cd-ing.

## Searching: check whether `grep` here is ugrep, not GNU grep
On machines where `grep` resolves to **ugrep** (common with Homebrew setups), it rejects syntax GNU grep (and `find`) tolerate. Prefer the **Grep tool** (or `rg`) for searches — they sidestep the gotchas entirely. If you must use inline shell `grep`: don't use `find`-style `! -path "*/x/*"` exclusions (ugrep reads `!` and the globs as file args → `!: No such file or directory`, exit 2 — use `rg`/Grep `--glob '!…'` instead), and don't write empty or brace alternation branches like `'{footer}|{children}'` (→ `empty (sub)expression`, exit 2 — brace-wrap is not an alternation literal; list the alternatives plainly).

# File paths in tools are literal — never shell-escape them
`Read`/`Write`/`Edit` `file_path` values go straight to the filesystem — no shell interprets them — so shell metacharacters must appear **verbatim**, never backslash-escaped. A Next.js route-group dir is `…/(app)/settings/page.tsx`, never `…/\(app\)/settings/page.tsx`: the backslashes become literal path characters, the file isn't found, and the call fails `File does not exist` (often retried — wasted calls). Only Bash `command` strings get shell escaping/quoting; tool path arguments never do. A PreToolUse guard (`bin/normalize-tool-path.py`, V-277) backstops this — it silently de-escapes a backslash-escaped shell metacharacter in a tool `file_path` when the de-escaped path exists and the as-given one doesn't — but the guard only fires on a path that resolves; write paths verbatim and don't lean on it.
